## Spec

### Problem / motivation

The legal & policy team uses three internal tools — Newsroom (media monitoring), First Pass (regulatory + ICTS analysis), and Canary (risk detection) — each living in its own GitHub repo, each with its own UI shell, its own Configure tab, its own Google login, and its own localStorage namespace. There's no shared home, no shared identity, no shared activity surface. The user wants Policy OS as that umbrella.

### What we're building

A single-page static HTML launcher at `~/Documents/legal/Policy-OS/` that:
- Lives alongside the three tool repos (Newsroom, First-Pass, Canary cloned to the same parent dir)
- Provides a 240px left sidebar as the workspace nav (Home + Tools + Workspace sections)
- Opens each tool **inside** the Policy OS shell via iframes (no page jumps)
- Surfaces a "Highlights" TLDR strip with mocked daily/weekly content per tool, above the fold
- Is fully brand-aligned to `BRAND_RULES.md` (Applied Sans, Applied Blue accents only, Tertiary palette for tool category coding, light-mode #F5F5F7 surfaces, max 8px radius, no shadows)
- Sets up the structural pattern for Option B (real shared shell with one auth, one Configure, cross-tool flows)

### User story

> As a member of the legal & policy team, I want to open one tab in the morning and immediately see what each tool surfaced overnight, so I can decide which one to dive into without context-switching between three browser bookmarks.

### Acceptance criteria

- [x] Three sibling repos (Newsroom, First-Pass, Canary) cloned to `~/Documents/legal/`

### Out of scope

- Cross-tool data flow (Canary → First Pass, etc.)

- A real `+ Add tool` flow (placeholder only)

### Decisions made

- **Sidebar over top-bar tabs** — sidebar scales to many tools, matches the underlying tools' own UI pattern, reads as workspace not settings.
- **Highlights TLDR over hero imagery** — imagery looked like marketing site; mocked-but-believable activity content earns the space and previews real value.
- **Persistent iframes + visibility toggle** (not swap-the-src) — all three tools mount once, navigation just toggles `.active` (display: block). Switching tabs is instant; each tool preserves scroll position and internal state. Demonstrates the shared-shell pattern without modifying any of the three underlying tools. Cosmetic double-sidebar is the only side-effect, fixed in Option B.
- **Direct paths to the actual tool HTML files** (`Newsroom.html`, `FirstPass.html`, `Canary.html`) — bypasses the meta-refresh redirect that each tool's own `index.html` does.
- **Applied Sans Display + Text + Mono** — using all three optical sizes per the brand font package; Display for headlines, Text for body, Mono for metadata.
- **Mid-tier Tertiary colors for tool category** — Newsroom = Dark Blue `#2635C8` (distinct from CTA-reserved Applied Blue), First Pass = Mid Green `#0BC242`, Canary = Mid Yellow `#FFBE32` (matches existing Canary brand).
- **Skip the Air/Land/Sea imagery in v0.1.0** — kept the PNG assets for future use (subtle empty states per brand rules), removed all usage from the launcher.

### Issues hit

- **Initial design was off-brand twice** — first dark mode + wrong blue (rebuilt to light mode + Applied palette), then DM Sans (rebuilt to Applied Sans). Lesson: read brand source-of-truth files before designing.
- **Brand rules incomplete** — `BRAND_RULES.md` was missing the Tertiary palette (the purple/dark-blue/green/yellow/orange ramps from the official Applied palette image). Added.
- **Two SECOND_BRAIN.md canonicals disagreed** — `~/.cursor/second-brain/SECOND_BRAIN.md` and `~/zoyatalkingtoherself/SECOND_BRAIN.md` each claim to be canonical and point at different "real" sources. Mirrored Policy OS additions to both; flagged consolidation as separate cleanup.
- **`file://` URL caching** — hard-refresh wasn't enough for font/CSS changes; incognito was the reliable nuke.
- **Hero imagery treatment took 3 iterations** — full-bleed header → smaller side panel → removed entirely. Brand rules even said imagery was for empty states; should have followed on first pass.
- **Cosmetic double-sidebar** in iframe view — Policy OS sidebar + tool's own sidebar both render. Filed in ISSUES.md, planned fix in Option B via `?embed=1`.
- **Git identity** — both commits attributed to `Zoya Hsiao <zoyahsiao@Zoyas-MacBook-Pro.local>` because global `user.name` / `user.email` aren't set. Flagged for user to fix once.
